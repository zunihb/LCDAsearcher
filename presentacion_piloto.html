01 — EL PROBLEMA

Encontrar sinergias entre 90 investigadores, a mano, es inviable

Revisar manualmente cientos de publicaciones por investigador para detectar temas en común toma horas por persona y el resultado queda obsoleto con cada nuevo paper. Con 90 perfiles son ~4.000 pares posibles de colaboración que nadie puede evaluar a ojo.

02 — CÓMO FUNCIONA

Pipeline automático en cinco pasos

Todo queda persistido en una base de datos consultable (SQLite): el grafo se reconstruye y actualiza cuando se quiera, y los datos sirven para cualquier análisis futuro.

Extracción

Se descargan perfil, métricas (índice h, citas) y todas las publicaciones desde Google Scholar.

Citas y coautores

Por cada paper: conteo de citas + papers citantes con sus autores → nuevos candidatos.

IA / NLP

Un LLM extrae 5 keywords por paper y estandariza términos ("PLL" = "phase-locked loop").

Cruce de datos

SQL detecta intersecciones: keywords compartidas, coautorías directas, citas cruzadas.

Visualización

Grafo interactivo HTML + tabla de sinergias + reporte automático.

Proceso manual

Leer cientos de títulos y abstracts por investigador, anotar temas, cruzar a mano contra cada otro perfil. Horas por persona, resultado estático y subjetivo.

vs.

Con el sistema

Se entrega la lista de IDs de Scholar y el pipeline genera grafo, tabla de sinergias y reporte. Minutos por investigador, repetible y siempre actualizable.

06 — VISIÓN A FUTURO

Del piloto a los 90 investigadores

La arquitectura del piloto ya está diseñada para escalar: agregar un investigador es agregar una línea de configuración.

a

Clustering automático de comunidades

El sistema agrupa a los 90 perfiles en sub-comunidades temáticas (ej. "Grupo de Potencia", "Control Automático", "Software") mediante algoritmos de detección de comunidades sobre el grafo.

b

Buscador interno de expertos

Una interfaz web donde se escribe "necesito un experto en análisis de Lyapunov" y el sistema responde quiénes han publicado sobre eso dentro del proyecto, ordenados por relevancia.

c

Red de citas: descubrir investigadores nuevos

Los autores que citan los papers del grupo ya quedan registrados en la base de datos — son candidatos naturales para ampliar la red de colaboración fuera del proyecto.

d

Mapa de calor de sinergias

Matriz 90×90 que colorea de un vistazo qué pares de investigadores tienen mayor solapamiento temático y aún no han colaborado: las oportunidades perdidas se vuelven visibles.

07 — ETAPA AVANZADA

Lectura inteligente de papers: del PDF al conocimiento consultable

Una capa donde el investigador sube el PDF de un paper y una IA de comprensión de documentos —muy superior al OCR clásico— extrae el texto, las ecuaciones en LaTeX y las tablas. Todo se indexa en un sistema RAG para responder preguntas sobre el contenido y afinar el grafo.

⬆

1 · Subida

El usuario carga el PDF del paper (propio, interno o publicado).

→
⌘

2 · Extracción IA

Modelos como Docling / Nougat (local) o Mistral OCR / LlamaParse (API) devuelven texto, LaTeX y tablas estructuradas.

→
▦

3 · Indexado RAG

El contenido se trocea, se vectoriza y se guarda en la misma base de datos (sqlite-vec).

→
✦

4 · Consulta

Se pregunta en lenguaje natural y el sistema responde citando el fragmento exacto del paper.

Keywords de texto completo

Hoy el piloto lee solo el abstract; con el PDF completo (métodos, ecuaciones) la detección de sinergias es mucho más precisa.

Respuestas, no solo nombres

Más allá de "quién sabe de X", responde "qué método usaron en tal paper" con la cita textual.

Privacidad opcional

El modo local procesa papers no publicados sin enviarlos a ninguna API externa.

08 — QUÉ SE NECESITA PARA PARTIR

Solo la lista de perfiles

El piloto con dos investigadores valida la metodología completa. Para escalar, basta con entregar los enlaces de Google Scholar (o nombres y afiliación) de los investigadores del proyecto. Cada perfil agregado enriquece el grafo y mejora la detección de comunidades.

scholar.google.com/citations?user=… o nombre + universidad o código ORCID
